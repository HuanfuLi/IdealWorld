# Plan 03-04: UI/UX Standardization & Cleanup (Wave 4)

## Goal
Harden the UI/UX consistency by tokenizing hardcoded colors, standardizing typography, and aligning CTA language across the platform as per the Phase 3 UI Review.

## Requirements
- **UI-01:** Tokenize ~87 hardcoded hex values into semantic CSS variables (index.css).
- **UI-02:** Standardize typography and remove inline styles in `PhysicsLaboratory.tsx`.
- **UI-03:** Standardize CTA labels across `SettingsPage.tsx` and `IdeaInput.tsx`.

## Implementation Tasks

### 1. Color Tokenization
1. **Define Core Tokens**: Update `web/src/index.css` to define a full set of semantic variables if they are missing (e.g., `--primary-emerald`, `--danger-red`, `--surface-glass`).
2. **Bulk Component Update**: Systematically audit and replace hardcoded hex values (e.g., `#10b981`, `#ef4444`) in `Simulation.tsx`, `TelemetryPanel.tsx`, and other components.
3. **Glassmorphism Alignment**: Ensure all glass panels use a consistent background/border token instead of local opacity overrides.

### 2. Typography Standardization
1. **Physics Lab Audit**: Identify all inline `fontSize` and `lineHeight` styles in `PhysicsLaboratory.tsx`.
2. **Tailwind Migration**: Replace inline styles with existing Tailwind utility classes (e.g., `text-sm`, `leading-tight`).
3. **Consistency Check**: Verify that typography scales match the rest of the application (e.g., headers in Simulation vs. Physics Lab).

### 3. Copy Standardization
1. **CTA Audit**: Compare button labels in `SettingsPage.tsx`, `IdeaInput.tsx`, and `HomePage.tsx`.
2. **Language Update**: Update generic labels (e.g., "Save Settings") to evocative, project-specific language (e.g., "Apply Configuration", "Initialize Idea").
3. **Tooltip Alignment**: Ensure tooltip/instructional text uses the same domain terminology as the CTAs.

## Verification Tasks

### Visual Audit
1. **Style Consistency**: Manually verify that all buttons and panels remain visually identical after tokenization.
2. **Typography Check**: Ensure `PhysicsLaboratory.tsx` maintains its hierarchy and readability across different viewport sizes.

### Code Audit
1. **Hex Count**: Run a grep command to verify that the number of hardcoded hex values in `web/src` has dropped to near-zero.
2. **Inline Style Check**: Verify no `style={{ fontSize: ... }}` remains in `PhysicsLaboratory.tsx`.

## Success Criteria
- [ ] 0% hardcoded hex colors in primary simulation and settings components.
- [ ] All typography in `PhysicsLaboratory.tsx` is utility-driven (Tailwind).
- [ ] CTA buttons across all entry pages use standardized, project-specific language.
