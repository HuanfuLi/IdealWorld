IDEAL WORLD: NEURO-SYMBOLIC ABM PROJECT MEMORY1. 项目愿景与核心架构 (Project Vision & Architecture)项目名称: Ideal World (基于神经-符号架构的社会经济模拟器)核心架构: Neuro-Symbolic Agent-Based Modeling (ABM)神经层 (Neuro Layer / LLM): 负责 Agent 的意图生成、自然语言处理、社会反思以及基于提示词（虚拟法律/道德设定）的策略选择。符号层 (Symbolic Layer / Physics Engine): TypeScript 编写的确定的物理与经济引擎。负责热力学代谢计算、疾病判定、订单撮合、做市商定价以及存量-流量一致性（SFC）校验。核心法则: 物理引擎决定了“能不能（Can）”，虚拟法律/LLM Prompt 决定了“想不想（Want）”和“该不该（Should）”。2. 架构演进与核心机制 (Evolution & Core Mechanics)2.1 混合微回合制与动作队列 (Hybrid Micro-Turn & Action Queue)时间尺度: 1 Iteration = 1 Week。多动作并发: 废除了单动作瓶颈，废除独立的 Action Parser。Agent 在一次 LLM 呼叫中，通过结构化 JSON 数组输出最多 3 个顺序执行的动作（ActionQueue）。顺序中断 (Interrupts): 物理引擎顺序结算动作。如果 Action 1 导致 Agent 健康极低或皮质醇极高，Action 2 和 3 将被强制阻断。2.2 基于经验数据的物理引擎 (Empirical Physics - The "Body")MET 代谢系统: 不同劳动消耗卡路里不同。重体力劳动（如 sys_farm）消耗 7.25x METs，认知劳动消耗 1.4x METs。引擎具备“Eat to Full（按需吃饱）”逻辑，防止高薪体力劳动者因每回合只能吃 1 个食物而“过劳死”。稳态负荷 (Allostatic Load): 瞬时压力（Cortisol）转化为可逆应变（Strain），再固化为不可逆负荷（Load），最终导致永久性健康衰减（Health Decay）。认知钳制 (Psychological Clamping): 物理约束心灵。最大幸福度受制于物理健康与皮质醇公式：Max_Happiness = 100 - (Cortisol * 0.5) - (100 - Health)，防止 LLM 在濒死时产生“极度幸福”的幻觉。2.3 宏观经济与流动性 (Macroeconomics & Liquidity)全品类 AMM (恒定乘积做市商): 彻底废除以物易物（Barter）。系统设立多商品资金池（$x \cdot y = k$），涵盖 Food, Tech_Parts, Luxury_Services。确保服务业在早期有流动性变现。Stock-Flow Consistent (SFC) UBI: 经济系统严格遵守存量-流量一致性。无印钞，无凭空销毁。AMM 的法币利润 + 财富滞纳金（Demurrage Tax）100% 注入 UBI 资金池，全额返还给全民，防止通货紧缩死锁。绝对自由市场: 取消商品价格上限，通过在 Prompt 中注入 [PROFIT ALERT]，依靠高昂的价格信号（暴利预期）引导 Agent 打破角色扮演锁定，自发转行生产短缺物资。3. 历史踩坑与修复记录 6. 本次架构研讨记忆 (Session Memory: Economic Engine Overhaul)
本轮对话核心分析了现有经济系统中私有企业发展的阻力与机械降神问题：
- **硬编码的无限金钱与物资 (Deus Ex Machina):** 目前的 `sys_farm` 和 `sys_factory` 是系统硬编码的“无底洞”，它们为了发工资凭空印发 Fiat，同时强行将其员工的产出塞进 AMM。这破坏了整个系统 SFC，导致严重的通货膨胀。
- **私企不能生产 (The Missing Production Link):** 审查代码发现，公民通过 `FOUND_ENTERPRISE` 和 `HIRE_EMPLOYEE` 创建私营企业并开工发工资后，**打工人的产出并没有结算到企业主的背包里**。这导致当老板成为了一个纯粹的“发钱做慈善”行为，使得模型中的经济社会无法向更高级的生产关系自然演进。
- **重构方案 (Phase 7 - Transition to Natural Economy):**
  1. 彻底删除 `simulationRunner.ts` 中的硬编码体制内企业（`sys_farm`, `sys_factory`）。
  2. 将初始经济资源、法币总量、税收或福利转由第一回合的**设计阶段（Design Phase）参数控制**（从 JSON 中读取系统初始设定）。
  3. **核心物理引擎修复**：更新 `WORK_AT_ENTERPRISE` 逻辑，员工消耗卡路里产出的物资，必须直接存入**企业主的 Inventory**。
  4. 企业主必须自主通过 `POST_SELL_ORDER` 将物资变现来发工资。这彻底打通了私有制经济闭环，并保证了全局法币的绝对守恒。
上述计划已作为 Phase 7 Plan 追加到了 `CONVERSATION.md` 中。
